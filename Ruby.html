Meet Ruby

Ruby was designed and developed starting in the mid-1990s by Yukihiro 'Matz' Matsumoto. Matz created Ruby to be productive and fun compared with other programming languages. He felt that the design of a programming language should revolve first around the human and second around the computer. Ruby is an object-oriented, dynamic, general-purpose, programming language that has a wide user base and a growing following.

Object Oriented

Ruby is an object-oriented programming language with classes that define object attributes and behaviors. Objects can inherit attributes and behaviors from parent objects, which is a powerful feature of object-oriented design.

Dynamic

Ruby is dynamic in that methods and objects can be manipulated and extended at run-time. Classes are never truly cemented down in Ruby as methods and attributes can be added while the program is running if necessary.

General Purpose

Ruby is a full-featured programming language that can be used in many situations. It is particularly useful as the server-side solution in web development.

Why Ruby?

Ruby seems to cause a lot of divisiveness in the development community. Some hail its advantages as the Second Coming of programming that solve all problems, whereas others complain that it has some serious deficiencies that make it not worth using at all.

Pros

Proponents of Ruby love its flexibility and balance. See the list below for a few of the best features of Ruby, including some examples.

  1. Everything is an object!
  2. Ruby is extremely flexible
  3. Blocks
  4. Mixins
  5. Lack of special characters
  6. Rails (fast and easy web app development)

Cons

Detractors of Ruby argue that the below items make it not worth using. (You can read this article for a Ruby lovers rebuttal to each of the bullets below).

  1. Ruby is not as mature as Java, PHP, etc.
  2. Ruby has subpar performance
  3. Ruby and Windows don't play nice together
  4. Ruby isn't as popular/common as PHP
  5. Ruby users are condescending and snobbish
  6. Ruby is too opinionated and forces developers to adopt designs
  7. Ruby is not as reliable as languages like Java and C#
  8. Ruby cannot handle the enterprise
  9. Ruby can't scale
  10. Skilled Ruby developers are hard to come by

Ruby in the Wild

Ruby is most often used in the wild with the web application framework Ruby on Rails. Rails is an open-source framework that was first released in 2003 and helps to make developing web applications a dreamier experience.

Companies Utilizing Ruby

Largely because of the aforementioned Ruby on Rails framework, Ruby is gaining popularity in the industry. Businesses can leverage the productivity that Rails offers to stand-up web applications extremely quickly, making Ruby an attractive choice for projects. Some notable companies that use Ruby:

  • Github
  • Twitter
  • Shopify
  • Money Desktop

Careers with Ruby

A quick search for the keyword 'Ruby' on Dice.com results in roughly 2300 available jobs. This is a fairly small number when compared to some other, more mature langauges, such as the following:

  • Java - 15000 jobs
  • C# - 8000 jobs
  • Python - 4000 jobs
  • PHP - 3000 jobs
There is not an over-abundance of jobs that make learning Ruby an absolute economic motivator, but it is growing and desired. Certainly having Ruby as an additional skill is attractive to any potential employers.
